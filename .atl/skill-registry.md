# AG-Ágil Gestión — Skill Registry

## Project Context
Sistema de gestión empresarial para pequeños negocios en Ecuador. Incluye punto de venta (POS), inventario, facturación electrónica SRI (XML XAdES-BES) y multi-tenancy.

## Tech Stack
- Frontend: Next.js 15 App Router + React 19 + Tailwind CSS 4 + Zustand 5
- Backend: Next.js API Routes + Drizzle ORM + PostgreSQL
- Validation: Zod 4
- Testing: Vitest (unit) + Playwright (E2E)
- Monorepo: pnpm + Turborepo

## Project Standards (compact rules)

### Next.js 15
- Use App Router exclusively
- Server Components by default, "use client" only when needed
- Server Actions for mutations, fetch in Server Components for reads
- route.ts for external webhooks/API

### React 19
- No useMemo/useCallback needed (React Compiler)
- Use simple patterns, avoid over-engineering
- Prefer Server Components for data fetching

### Tailwind 4
- Use theme variables in global CSS, NOT var() in className
- Mobile-first responsive design
- cn() utility for class merging (clsx + tailwind-merge)

### TypeScript
- Strict mode enabled
- Define explicit types in packages/shared
- Avoid `any` — use `unknown` for truly unknown types

### Zod 4
- v4 breaking changes from v3 apply
- Use in shared DTOs and API validation layers

### Zustand 5
- Keep stores granular (one per domain concept)
- Persist middleware for offline support

### Clean Architecture
- Domain layer has ZERO external dependencies
- Infrastructure depends on Domain, never the reverse
- UI depends on shared types, never on Infrastructure directly
